1、数据库概述
数据库（database）：以特定的结构批量存储业务数据的软件。
网状数据库、层次数据库、关系型数据库、非关系型数据库
关系型数据库逻辑结构（RDBMS）：
服务器（server）->
数据库（database）->
表格（table）->
行（row）->
列（column）
常见的关系型数据库
SQLite-微型数据库，常用语移动设备
MySQL-开源的中小型数据库，可用于各种操作系统
PostgreSQL-开源中小型数据库
SQL Server-Microsoft开发的中型数据库，只用于Windows系统
Oracle-Oracle公司开发的中大型数据库，可用于各种操作系统

4.MySQL常用的SQL命令
SQL：Structured Query Language,结构化查询语言，用于对关系型数据库服务器中的数据进行增删改查的操作
SQL语言最早是由IBM提出，后提交给了ISO，最终成了数据库行业的标准语言，分为SQL-87，SQL-92，SQL-99等，
当前标准的SQL命令可以被大多数的关系型数据库所支持
SQL命令的两种执行方式
（1）交互模式：客户端输入一行，点击回车，服务器执行一行。适用于临时性的查看数据 mysql -uroot 回车
（2）脚本模式：客户端把要执行的命令写在一个文本文件中，一次性的提交个服务器执行。适用于批量的增删改查
mysql -uroot < 直接把脚本文件拖进去 回车

1.计算机如何存储字符
（1）如何存储英文字符
ASCII：总共有128个 ，对所有的英文字符以及标点符号进行了编码。
Latin-1：总共有256个，兼容ASCII码，同时对欧洲符号进行编码。Mysql就默认使用这用编码。
（2）如何存储中文字符
GB2312 对常用的6千多汉字进行了编码，兼容ASCII编码
GBK 对2万多汉字进行了编码，兼容GB2312
BIG5 台湾繁体字编码，兼容ASCII
Unicode 对世界上主流的语言常用的字符进行了编码，兼容ASCII，不兼容GBK,GC2312，BIG5等系列
具体分为UTF-8，UTF-16，UTF-32等存储方案。
(3)解决MySQL储存中文乱码
sql脚本文件储存的编码为utf-8
客户端连接服务器端使用的编码为utf-8
服务器端创建数据库储存使用的编码为utf-8
2.MySQL中的列类型
创建数据表的时候，指定的列可以存储的数据类型。
CREATE TABLE book（bid 列类型）
(1)数值类型——引号可加可不加
tinyint(微整型)占1个字节(BT)范围-128到127
smallint 小整型 占2个字节 范围-32768到32767
int 整型 占4个字节 范围-2147483648到2147483647
bigint大整型 占8个字节
浮点型
float 单精度浮点型 占4个字节 范围3.4e38，范围比int大的多，可能产生计算误差。
double 双精度浮点学，占8个字节 范围比bigint大的多
decimal(M,D)定点小数，不会产生计算误差，M：总的有效位数，D：小数点后的有效位数
bool 布尔型 只有两个结果 true/1，false/0，true和false不能添加引号，正真在存储数据的时候，会使用tinyint
（2）日起时间类型—–必须加引号
date 日起型 ‘2018-12-31’
time 时间型 ‘14:20:30’
datetime 日起时间型 ‘2013-15-23 12:30:15’
（3）字符串类型
varchar(M)变长字符串 不会产生空间的浪费，操作时间相对较慢，M最大值是65535
char(M) 定长字符串 可能产生空间浪费，造作速度较快，M最大值是255；用于储存一个人的手机号等固定长度字符。
text(M) 大型变长字符串，最多存2G
3.列约束
MySQL可以对插入的数进行特定的验证，只有满足条件才允许插入到数据表中，否则被认为是非法的插入。
（1）主键约束—primary key
声明了主键列上的值不能出现重复，表中查询的记录会按照，主键由小到大排序–加快查找速度；通常主键添加到编号列中。
注意:一个表中只能有一个主键。
（2）唯一约束—unique
声明了唯一约束的列上不能插入重复的值，允许插入NULL，并且能够插入多个NULL。
NULL =空 ：在插入数据中，暂时无法确定要保存的数据 null 注意NULL不能加’’
(3)非空约束—not null
声明了非空约束的列上不能插入null
(4)默认值约束—default
可以使用default关键字声明默认值，有两种方式可以应用默认值
insert into xz values(40,’苹果’,default)
insert into xz(fid,fname) values(20,’华硕’)
（5）检查约束—check
检查约束可以对插入的数据进行检验
create table student(age int check(age>=18 and age<=60));MySQL不支持检查约束，会降低数据的插入速度。
（6）外键约束—foreign key
声明了外键约束的列，取值必须在另一个表的主键列上出现过，列类型要保持一致，取值可以是null。
foreign key(familyId)references:引用xz_lapyop_family(fid)
4.MySQL中的自增列
auto_increment:自动增长，假如一个列声明了自增列，无需手动赋值，直接设置为null，会获取当前最大值，然后加1插入。
注意：
自增列允许自动赋值
只适用于整数型的主键列上
5.简单查询
（1）查询特定的列
示例:查询员工的姓名，工资，生日
select ename,salary,birthday from emp;
(2)查询所有的列
select from emp;
(3)给列起别名
示例：查询所有的员工的姓名和工资，使用汉字的别名显示
select ename as 姓名,salary as 工资 from emp;
注意:as 关键字可以省略 但是要保留空格
(4)只显示不同的记录/合并相同的记录
示例：查询出员工都在哪些部门
select distinct deptId from emp;
(5)在查询时进行计算
示例：计算23/4-8+10
select 23/4-8+10;
练习：每个月加500，奖金5000
select ename 名字 ,(salary+500)12+5000 年薪 from emp;
(6)查询结果集的排序 select from … order by…
结果按照部门编号升序排序 asc向上
select from dept order by did asc;
示例：结果按照部门编号降序排序 desc 降序
select from dept order by did desc;
order by 可以按照数值，日期/时间，字符串来排序 默认按照升序排列
(7)条件查询
示例：查询出编号为5的员工所有信息
select from emp where eid=”5”;
select from emp where salary>=5000 and salary<=7000; 工资在5000~7000
select from emp where salary<5000 or="" salary="">7000; 工资小于5000或者大于7000
select from emp where deptId=10 or deptId=30;
select from emp where deptId not in(10,30);
is null/is not null and/or between…and…/not between..and… in()/not in()
注意：删除，更改，查询都可以结合条件查询
(8)模糊条件查询
示例：查询出员工姓名含有字母e的所有信息 关键字like’%….%’
select from emp where ename like’%S%’;
练习： 查询出名字中以’e’结尾的
select from emp where ename like’%Q’;
练习：查出倒数第二个以e结尾的
select from emp where ename like’%P_’;
sql中提供了两个模糊查询的字符
% 可以匹配任意多个字符 >=0
_ 可以匹配任意一个字符 =1
注意：以上两个匹配不能和=使用，必须使用LIKE关键字
(9)分页查询
假如查询的结果集中有太多的数据 ，一次显示不完，可以分页显示。
需要有两个条件： 当前的页码 每页的数据量 关键字limit
select from emp limit start,conut;
start: 是一个数字，从结果集中的哪一条开始读取，默认的第一条称为0
count: 是一个数字，最多读取的行数
start=（页码-1）count
select from emp limit 0,5; 第一页每一页显示5条记录
select from emp limit 5,5; 第二页
select from emp limit 10,5; 第三页
6.复杂查询—聚合查询/分组查询
示例：查询出所有员工的数量 关键字 count() 最好用count()
select count(eid) from emp;
函数：功能体，接受若干个数据，返回特定的结果 聚合函数count()所有/sum()求和/avg()平均/max()最大/min()最小
示例：求出所有员工工资总和
select sum(salary) from emp;
示例：所有男员工的平均工资
select avg(salary) from emp where sex=1;
分组查询:只能查询分组条件和聚合函数
示例：查询出每个部门员工最高工资 关键字 group by
select max(salary) from emp group by deptId;
练习：男女员工的平均工资，最高工资，最低工资
select sex, avg(salary),max(salary),min(salary)from emp group by sex;
year()获取日期中的年份
month()获取日期中的月份
示例：查询1991年出生的员工
select from emp where year(birthday)=’1991’;
示例：查询3月出生的员工
select from emp where month(birthday)=’3’;
复杂查询—子查询
一个SQL语句的查询结果作为另一个SQL语句的查询条件
示例：查询出研发部员工的所有信息
步骤1：查询研发部的部门编号
select did from dept where dname=”研发部”;
步骤2：使用编号查询员工
select from emp where deptId=10;
综合到一起：select from emp where deptId=(select did from dept where dname=”研发部”);
select salary from emp where ename=’小王’;
select from emp where salary>(select salary from emp where ename=’小王’);
select * from emp where year(birthday)=(select year(birthday) from emp where ename=’小王’);
复杂查询—多表查询
示例：查询出所有员工的姓名及其所在的部门名称
!!!select ename,dname from emp,dept;
如何避免笛卡尔积：为多表查询添加条件。
select ename,dname from emp,dept where deptId=did;
上述语法无法查询出没有部门的员工，也不能查询出没有员工的部门。
解决办法:
(1)内连接：inner join…on—和上面的结果一样
select ename,dname from emp inner join dept on deptId=did;
（2）左外链接 left outer join…on 查询结果是左侧表中所有记录，即使右侧表中没有对应的记录
select ename,dname from emp left outer join dept on deptId=did; outer可以省略
（3）又外链接 right outer join…on outer可以省略
select ename,dname from emp right join dept on deptId=did; 查询结果是右侧表中所有记录，即使左侧表中没有对应的记录
（4）全链接：full join 】显示左右两侧表中的所有记录—-MySQL不支持
MySQL中的全链接： 左外链接union右外链接，合并相同的记录
左外链接union all右外链接 ，不合并相同的记录
